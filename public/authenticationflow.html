Authentication Flow

Overview

The authentication flow begins after a valid chat server is discovered using the DiscoveryManager. Once the server metadata (i.e., a DiscoveryPayload) is available, the login process can begin.

The goal of this flow is to obtain an access token from the authentication server (e.g., Keycloak), which is later passed to the ClientManager for connecting to chat services.

Flow Summary

  • The user selects or enters a discovery URL
  • The DiscoveryManager emits discoverySuccess with a DiscoveryPayload
  • The application pushes AuthBrowserPage onto the stack and passes the payload
  • AuthManager is used to initiate the login process
  • Internally, AuthCore manages the OAuth2 protocol and emits a URL to be shown
  • AuthBrowserPage displays the login page in a WebEngineView
  • When login succeeds, AuthManager emits authenticationSuccess with the access token
  • The app pops the AuthBrowserPage and proceeds to connect using ClientManager

Components Involved

  • DiscoveryPayload
  • AuthManager
  • AuthCore
  • AuthBrowserPage

See also AuthManager, AuthBrowserPage, ClientManager, and discoveryflow.html.
